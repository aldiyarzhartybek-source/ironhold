# IronHold — Stage 1 (MVP Gameplay Loop)

Формат: Название карточки / Описание / Критерии приёмки

## 1) Реализовать RuntimeLevelState и старт волны

### Описание
Добавить рабочее состояние уровня: текущая волна, таймеры спавна, статус
(IDLE, RUNNING, COMPLETED, FAILED), базовые счётчики.

### Критерии приёмки
- Уровень корректно переходит из IDLE в RUNNING.
- Данные волны и спавна читаются из WaveDefinition.
- Статус уровня обновляется без падений.

## 2) Сделать спавн врагов из конфигов волн

### Описание
Реализовать последовательный спавн врагов по waves.json с учётом enemyId,
count, spawnIntervalSec.

### Критерии приёмки
- На карте появляются враги согласно конфигу волны.
- Частота и количество спавна соответствуют настройкам.
- После окончания волны новых врагов не появляется.

## 3) Добавить движение врагов по пути карты

### Описание
Реализовать перемещение врагов от старта к финишу по данным карты
(или временным waypoint'ам), с удалением при достижении конца.

### Критерии приёмки
- Враги движутся предсказуемо от точки A к B.
- Скорость учитывает параметр speed из Enemy.
- При достижении конца враг удаляется и обновляет состояние уровня (жизни/штраф).

## 4) Реализовать BuildSlot interaction и установку башни

### Описание
Сделать кликабельные BuildSlot и установку башни из базового списка
(минимум 1 тип).

### Критерии приёмки
- По клику на свободный слот ставится башня.
- Повторная установка в занятый слот блокируется.
- Установленная башня отображается на экране.

## 5) Подключить экономику постройки/награды

### Описание
Интегрировать EconomyState: списание золота за строительство, награда за
убийство, отказ при нехватке золота.

### Критерии приёмки
- При успешной постройке золото уменьшается на cost.
- При убийстве врага начисляется награда с учётом multiplier.
- При нехватке золота постройка не выполняется.

## 6) Реализовать таргетинг и стрельбу башни

### Описание
Добавить минимальный боевой цикл: башня ищет цель в радиусе и наносит урон
с учётом fireRateSec.

### Критерии приёмки
- Башня выбирает врага в зоне действия.
- Урон наносится с корректным интервалом.
- HP врага уменьшается до 0, после чего враг удаляется.

## 7) Добавить HUD игрового экрана (Stage UI)

### Описание
Показать текущие ключевые метрики: золото, волна, активные враги, жизни базы,
статус уровня.

### Критерии приёмки
- HUD обновляется в реальном времени.
- Значения синхронизированы с фактическим состоянием фасада.
- UI не ломает рендер карты/юнитов.

## 8) Доработать EventBus интеграцию в геймплее

### Описание
Подключить реальные игровые события (EnemySpawned, EnemyKilled, TowerBuilt,
WaveStarted, WaveCompleted) и обработчики.

### Критерии приёмки
- Есть как минимум 3 события, реально проходящие publish -> subscribe.
- Подписки/отписки работают без утечек слушателей.
- EventBus используется между системами, а не привязан к одному экрану.

## 9) Расширить GameFacade до orchestration-уровня

### Описание
Собрать в фасаде доступ к ключевым runtime-системам: уровень, экономика,
спавн, бой, слоты/башни.

### Критерии приёмки
- GameScreen работает через фасад, без хаотичного доступа к системам.
- Основные lifecycle-операции уровня доступны из фасада.
- Фасад остаётся расширяемым и не содержит UI-логики.

## 10) Добавить состояния завершения уровня (win/lose)

### Описание
Реализовать условия победы/поражения и реакцию UI (минимум текст/кнопка
restart/back to menu).

### Критерии приёмки
- Победа: все волны завершены и врагов не осталось.
- Поражение: исчерпаны жизни базы (или эквивалентное условие).
- После завершения уровня пользователь видит понятный результат.

## 11) Баланс и валидация конфигов Stage 1

### Описание
Подобрать рабочие значения для MVP и добавить базовую валидацию конфигов
(диапазоны/обязательные поля).

### Критерии приёмки
- Игровая петля проходима и не ломается на дефолтных конфигах.
- Некорректные значения в конфиге не приводят к крэшу.
- Есть fallback/лог предупреждения для проблемных параметров.
